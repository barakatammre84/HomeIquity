1️⃣ WHAT A “CHANGE OF CIRCUMSTANCE” IS (IN SYSTEM TERMS)

A Change of Circumstance (COC) is not “anything changed”.

It is:

A change that is material enough under the applicable investor policy to require re-review, re-disclosure, or invalidation of a pre-approval.

Your system detects deltas, then checks materiality thresholds.

2️⃣ CORE COC DATA MODEL (STABLE, CODED ONCE)
ChangeOfCircumstanceRule {
  dimension            // income, credit, assets, employment, property
  eventType             // decrease, increase, change, inquiry, etc.
  metric                // %, points, boolean, count
  thresholdRef          // policy-controlled
  severity              // NONE | REVIEW | MATERIAL
  action                // IGNORE | FLAG | INVALIDATE
  explanationTemplate
}


⚠️ Developers hardcode the engine, not the values.

4️⃣ HOW THE ENGINE USES THIS (VERY IMPORTANT)

Applicable COC rules

Determine severity

Take action

Example

Income dropped 9%

Fannie: REVIEW → Flag LO

Freddie: MATERIAL → Auto-invalidate

Same borrower, different investor.
That’s why this must be policy-driven.
