HOMIQUITY LENDER‑READY BORROWER PACKAGE
1. Borrower Overview
Purpose: establish identity and household context immediately.

Borrower name(s)

Household composition

Primary residence state

Borrower type

W‑2

Self‑employed

Mixed income

Investor

Short, factual, no narrative.

2. Transaction Intent
Purpose: clarify why the file exists.

Transaction type

Purchase

Refinance

Cash‑out

Property intent

Primary residence

Second home

Investment

Target timeframe (if provided)

No assumptions. No recommendations.

3. Declared Income Summary
Purpose: give underwriting a fast income map before documents.

Structured table format:

Income Source	Type	Frequency	Documentation Status
Employer A	W‑2	Bi‑weekly	Uploaded
Business B	Self‑employed	Monthly	Pending
Rental Property	Passive	Monthly	Uploaded
No calculations. No eligibility language.

4. Asset Overview
Purpose: show liquidity and sourcing categories.

Checking / savings accounts

Investment accounts

Retirement accounts

Other declared assets

Each listed as:

Category

Declared presence

Documentation status

No balances summarized unless verified.

5. Property Context (if applicable)
Purpose: anchor underwriting to property reality.

Property address (if identified)

Estimated purchase price or value (user‑provided)

Occupancy intent

No valuation claims.

6. Document Inventory
Purpose: eliminate back‑and‑forth.

Structured checklist:

Document Type	Status	Notes
Pay stubs	Uploaded	Most recent
W‑2s	Uploaded	Last 2 years
Bank statements	Uploaded	All pages
Tax returns	Missing	Required for self‑employed income
This is one of the most valuable sections for lenders.

7. Validation Notes
Purpose: flag issues early without judgment.

Document recency checks

Completeness checks

Consistency observations

Example:

“Bank statements uploaded include all pages and are within required date range.”

No conclusions. No risk language.

8. Readiness Status
Purpose: set expectations clearly.

Intake status: Complete / In Progress

Document status: Complete / Partial

Package status: Ready for underwriting review / Pending items

This is procedural, not evaluative.

9. Audit Trail
Purpose: compliance confidence.

Intake started date

Last update date

Document upload timestamps

Validation timestamps

This matters more than most founders realize.

10. Disclosure
Purpose: protect everyone.

“This package is informational and prepared to support underwriting review. It does not imply approval, eligibility, or loan terms.”

Why lenders love this format
Predictable structure

Fast to scan

No fluff

No compliance risk

Reduces clarification emails

Drops cleanly into LOS workflows

This is what justifies premium pricing for borrower handoffs.
