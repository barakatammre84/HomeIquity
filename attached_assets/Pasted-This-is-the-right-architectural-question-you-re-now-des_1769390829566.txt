🧩 THE OFFER BRIDGE ARCHITECTURE

(Broker-Controlled Shopping, Lender-Visible Acceptance)

1️⃣ CORE PRINCIPLE (DO NOT VIOLATE)

You control eligibility.
Lenders control pricing.
Clients control selection.

Your system orchestrates — it does not decide credit or pricing.

2️⃣ THE OFFER OBJECT (FOUNDATIONAL DATA MODEL)

Every “deal” is an Offer tied to a frozen eligibility snapshot.

Offer Object (Conceptual)

Offer ID

Lender ID

Product ID

Eligibility Snapshot ID

Rate

Points

Fees

Lock term

Expiration

Conditions

Broker comp (fixed, hidden until LE)

Status (Proposed / Selected / Locked / Expired)

This object is what everything else talks to.

3️⃣ HOW LENDERS FEED OFFERS (3-TIER STRATEGY)
🥇 TIER 1 — MANUAL (MVP / DAY 1)

Best for launch. Zero friction.

How it works:

Lender sends:

Rate sheet

Product matrix

Your Ops team:

Uploads or maps it

System auto-matches eligible borrowers

UI for Lenders:

None required

Email + portal upload is fine

✅ Fast
✅ Lender-friendly
❌ Less real-time

🥈 TIER 2 — SEMI-AUTOMATED (BEST PRACTICE)

This is your sweet spot for scale.

Lender Pricing Portal (Read-Only + Upload)

Upload rate sheets (CSV / Excel)

Set:

Products

Credit tiers

LLPAs

Lock terms

Validity windows

System:

Normalizes pricing

Applies to eligible files

Generates offers automatically

✅ Scalable
✅ No lender dev work
✅ Most lenders will use this

🥉 TIER 3 — FULL API (ENTERPRISE)

Only after volume + trust.

Lender API provides:

Real-time pricing

Lock confirmations

Product eligibility

Your system:

Calls API using eligibility snapshot

Receives offers

Displays to borrower

❌ Slow to onboard
❌ High lift
✅ Real-time

👉 Do not start here.

4️⃣ CLIENT OFFER UI (VERY IMPORTANT)
Borrower View

3–5 normalized offers max

Labels:

Lowest payment

Lowest total cost

Fastest close

Points toggle (on/off)

Break-even calculator

No lender chaos.

5️⃣ BROKER CONTROL PANEL (YOU)

You can:

Suppress lenders

Rank offers

Apply house overlays

Set comp once

All actions logged.

6️⃣ LENDER VISIBILITY (TRUST BUILDER)
When client selects an offer:

Lender sees:

Eligibility snapshot

Offer accepted

Timestamp

No changes pending

This prevents:

Re-shopping

Bait-and-switch

Lock disputes

7️⃣ LOCK & COMMIT FLOW
Lock Request Includes:

Offer ID

Snapshot ID

Client attestation

Change-of-circumstance check

Lender confirms lock:

Via portal

Or email

Or API (later)

Credit

Income

Assets

System:

Invalidates offers

Requires re-shop

This is HUGE for trust.
