# 1️⃣ DTI + DSCR ENGINE

## (Self-Employed, W-2, Rentals, Multi-Property)

> **Design principle:**
> 📌 *DTI is calculated from normalized income streams, not documents.*

Documents → OCR → Income Streams → **DTI / DSCR**

---

## I. CANONICAL DEFINITIONS (LOCK THESE NOW)

### DTI (Debt-to-Income)

```
DTI = Total Monthly Liabilities / Total Qualifying Monthly Income
```

### DSCR (Debt Service Coverage Ratio)

```
DSCR = Net Rental Income / Property PITIA
```

---

## II. INCOME NORMALIZATION (MOST IMPORTANT PART)

### IncomeStream Structure (recap)

```ts
IncomeStream {
  type
  monthlyAmount
  stability
  sourceDocuments[]
}
```

⚠️ **DTI logic never touches raw documents**

---

## III. W-2 INCOME LOGIC

### Step 1: Detect Income Type

```ts
IF paystub detected
AND employer consistent
→ IncomeType = W2
```

### Step 2: Calculate Base Income

| Pay Frequency | Formula                    |
| ------------- | -------------------------- |
| Hourly        | rate × avg hours × 52 / 12 |
| Salary        | annual salary / 12         |
| Biweekly      | gross × 26 / 12            |
| Semimonthly   | gross × 24 / 12            |

---

### Step 3: Bonus / OT / Commission

```ts
IF bonus OR OT detected
AND history ≥ 24 months
AND trending stable
→ include averaged income
ELSE
→ exclude
```

**Averaging**

```
(Bonus Year 1 + Bonus Year 2) / 24
```

---

### Step 4: Stability Flag

```ts
IF employer < 12 months
→ unstable
ELSE stable
```

---

## IV. SELF-EMPLOYED INCOME LOGIC (THIS IS WHERE SYSTEMS FAIL)

### Step 1: Determine Self-Employment

```ts
IF Schedule C OR K-1 OR Business Return detected
→ SelfEmployed = true
```

---

### Step 2: Extract Base Income

#### Schedule C Example

```
Net Profit
+ Depreciation
+ Depletion
+ Amortization
- Meals disallowance
= Qualifying Income
```

#### Schedule E (Rental)

Handled separately (see DSCR)

---

### Step 3: Two-Year Averaging

```ts
IF income declining
→ use lower year
ELSE
→ average
```

---

### Step 4: Business Debt Pass-Through

```ts
IF business liabilities exist
AND not already on credit report
→ add to liabilities
```

---

## V. RENTAL INCOME LOGIC (DTI SIDE)

### Step 1: Identify Rental Properties

```ts
IF Schedule E detected
OR Lease Agreement detected
OR Mortgage Statements > 1
→ Rental Property
```

---

### Step 2: Rental Income Calculation (Conventional)

```
Gross Rent
× 75%
- PITIA
= Net Rental Income
```

Rules:

* Negative → liability
* Positive → income

---

### Step 3: FHA / VA Rental Adjustments

* FHA often requires full PITIA offset
* Vacancy factor rules differ

⚠️ Engine must support **product-specific logic flags**

---

## VI. DSCR LOGIC (INVESTOR / HELOAN / NON-QM)

### Step 1: Property-Level Calculation

```ts
DSCR = Net Rent / PITIA
```

Where:

```
Net Rent = Lease Rent OR Market Rent
```

---

### Step 2: Qualification Rules

| DSCR      | Result   |
| --------- | -------- |
| ≥ 1.25    | Strong   |
| 1.00–1.24 | Marginal |
| < 1.00    | Fail     |

---

### Step 3: Multi-Property DSCR

```ts
PortfolioDSCR = Σ Net Rent / Σ PITIA
```

---

## VII. LIABILITY AGGREGATION

### Include:

* Mortgages
* HELOCs
* Auto
* Student loans
* Credit cards (min payment)
* Business debt passthrough

### Student Loans

```ts
IF payment reported
→ use actual
ELSE
→ use % of balance (product rules)
```

---

## VIII. HOUSING EXPENSE (FRONT-END DTI)

```ts
Housing = PITI + HOA + MI
```

Used for:

* FHA / VA qualification
* Ability-to-repay tests

---

## IX. FINAL DTI CALCULATION

```ts
DTI = Total Monthly Liabilities / Total Monthly Qualifying Income
```

---

## XI. CONFIDENCE SCORING

```ts
confidence =
  incomeConfidence
  × docConfidence
  × stabilityScore
```

Rules:

* ≥90% → auto-approve
* 80–89% → LO review
* <80% → manual underwriting
