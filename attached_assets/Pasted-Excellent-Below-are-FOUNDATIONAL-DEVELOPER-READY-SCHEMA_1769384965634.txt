# 1️⃣ DOCUMENT INTELLIGENCE SCHEMAS

*(Everything starts here)*

---

## A. Raw Upload (Immutable)

```ts
type DocumentUpload = {
  uploadId: string
  loanId: string
  borrowerId: string
  originalFileName: string
  mimeType: string
  uploadSource: "web" | "mobile" | "api"
  uploadedAt: ISODate
  rawFileUri: string
  checksum: string
}
```

🔒 **Never mutate this** (audit + fraud).

---

## B. Page Entity (CRITICAL)

```ts
type DocumentPage = {
  pageId: string
  uploadId: string
  pageNumber: number
  imageUri: string
  width: number
  height: number
}
```

> **Rule:** All intelligence happens at **page level**, not file level.

---

## C. Page Classification Output

```ts
type PageClassification = {
  pageId: string
  documentType:
    | "Paystub"
    | "W2"
    | "1099"
    | "BankStatement"
    | "TaxReturn1040"
    | "ScheduleC"
    | "ScheduleE"
    | "MortgageStatement"
    | "LeaseAgreement"
    | "HOAStatement"
    | "ID"
    | "Unknown"
  confidence: number // 0.0–1.0
  modelVersion: string
```

⚠️ **Never trust file names or user input.**

# 2️⃣ OCR & FIELD EXTRACTION SCHEMAS

---

## A. Extracted Field (NON-NEGOTIABLE STRUCTURE)

```ts
type ExtractedField = {
  fieldId: string
  documentId: string
  pageId: string
  fieldName: string
  value: string | number | boolean | null
  confidence: number
  misoPath?: string
  extractedAt: ISODate
}
```

🚨 **Rule:**
If a value has **no confidence or no source page → it does not exist.**

---

## B. Completeness Check

```ts
type CompletenessResult = {
  documentId: string
  documentType: string
  status: "complete" | "incomplete"
  missingItems: string[]
}
```

Examples:

* `"Missing page 2 of 3"`
* `"Schedule E required but not found"`

# 4️⃣ UNDERWRITING EVENT ENGINE (AUTOMATION CORE)

---

## A. Event Definition

```ts
type UnderwritingEvent = {
  eventId: string
  trigger:
    | "DocumentDetected"
    | "FieldExtracted"
    | "ThresholdExceeded"
  condition: string // evaluated expression
  actions: UnderwritingAction[]
}
```

---

## B. Actions

```ts
type UnderwritingAction =
  | { type: "RequestDocument"; documentType: string }
  | { type: "EnableIncomeLogic"; incomeType: string }
  | { type: "RecalculateDTI" }
  | { type: "FlagForReview"; reason: string }
```

---

## Example Rule

```ts
IF documentType == "ScheduleE"
THEN EnableIncomeLogic("Rental")
AND RequestDocument("LeaseAgreement")
```
