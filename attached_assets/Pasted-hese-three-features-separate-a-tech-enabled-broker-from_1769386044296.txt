1️⃣ CONFIDENCE SCORE (AGENT-VISIBLE, BORROWER-HIDDEN)
🎯 PURPOSE

Give agents signal, not decisions.

Agents want to know:

“Is this solid?”

“Will this fall apart?”

“How hard can I push the offer?”

Borrowers should not see this — it creates confusion and liability.

A. CONFIDENCE SCORE MODEL (NOT A SINGLE NUMBER)

Do not show one opaque score.
Show decomposed confidence so agents understand risk.

ConfidenceScore {
  overall: number            // 0.00 – 1.00
  income: number
  assets: number
  credit: number
  documentation: number
  stability: number
  riskFlags: string[]
}

Example
{
  "overall": 0.91,
  "income": 0.88,
  "assets": 0.95,
  "credit": 0.92,
  "documentation": 0.90,
  "stability": 0.85,
  "riskFlags": ["Variable bonus income"]
}

B. AGENT UI (CRITICAL)
Display as:

🟢 Strong (≥ 0.90)

🟡 Solid w/ Review (0.80 – 0.89)

🔴 Fragile (< 0.80)

Tooltip Language (Approved by Counsel)

“Confidence score reflects the strength and completeness of the loan file based on preliminary analysis. It is not a lender decision.”

C. WHY THIS IS POWERFUL

✔ Agents write stronger offers
✔ Fewer “hope-based” pre-approvals
✔ You reduce fallout dramatically
✔ No borrower confusion
✔ Zero lender liability

2️⃣ AUTO-EXPIRATION + CREDIT RE-PULL WORKFLOWS

This is mandatory if you want to operate at lender quality.

A. EXPIRATION RULES (STANDARDIZED)
ExpirationPolicy {
  preApprovalValidityDays: 90
  creditValidityDays: 120
  incomeDocValidityDays: 60
  assetDocValidityDays: 60
}


These are configurable per product.

B. AUTOMATED EXPIRATION ENGINE
Daily background job:
for each activePreApproval:
  if now > expirationDate:
    set status = EXPIRED
    notify borrower + LO + agent


No human involvement.

C. AUTO RE-PULL DECISION TREE (IMPORTANT)
Soft Pull (Default)

Triggered when:

Credit nearing expiration

No change in liabilities detected

Hard Pull (Only If)

Required for AUS submission

Lender selection finalized

Borrower re-consents

CreditRefreshDecision {
  type: "SOFT" | "HARD"
  reason
  consentId
}

D. BORROWER-FACING LANGUAGE (SAFE)

“Your pre-approval has expired and needs to be refreshed to ensure accuracy. This helps protect you and ensure lender acceptance.”

No panic language.

E. SNAPSHOT RULE (IMPORTANT)

Any refresh:

Creates new underwriting snapshot

Invalidates prior pre-approval letter

Generates new letter + disclaimers

Never mutate old decisions.

3️⃣ LENDER-SPECIFIC PRE-APPROVAL FORMATS (HUGE BROKER EDGE)

This is how you win agent trust and lender speed.

A. CORE IDEA

Same underwriting decision.
Different presentation per lender.

Never re-underwrite — just reformat.

B. LENDER FORMAT DEFINITION
LenderPreApprovalFormat {
  lenderId: string
  logo: string
  requiredFields: string[]
  disclaimerOverrides?: string[]
  brandingRules: object
}

C. EXAMPLE: LENDER A VS LENDER B
Lender A Requires:

Max loan amount

DTI disclosed

Asset reserves shown

Lender B Requires:

No DTI shown

Short-form letter

Strong disclaimer

Your system:

Filters data

Preserves snapshot reference

Only:

Language

Ordering
