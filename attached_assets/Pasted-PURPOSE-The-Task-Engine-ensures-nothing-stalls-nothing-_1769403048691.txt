PURPOSE

The Task Engine ensures nothing stalls, nothing is forgotten, and humans only work exceptions.

It does this by:

Auto-creating tasks

Assigning ownership

Enforcing SLAs

Escalating automatically

Logging everything for compliance

1️⃣ TASK CREATION (EVENT-DRIVEN, NOT MANUAL)

Tasks are never created manually by staff unless explicitly allowed.

Core Task Triggers
A. Document Intelligence Triggers
Event	Task
OCR missing page	“Request missing page 2 of bank statement”
Unreadable doc	“Request clearer document image”
Expired doc	“Request updated pay stub”

📌 Generated after OCR + classification, not upload.

B. Change-of-Circumstance Triggers
Change Detected	Task
Credit score drop	“Re-evaluate credit eligibility”
New tradeline	“Assess new liability impact”
Income variance	“Recalculate income stability”
Bank balance change	“Confirm asset sufficiency”

📌 These tasks may freeze the file automatically until resolved.

C. Workflow State Triggers
State Transition	Task
Intake → Docs	“Complete initial document set”
Verified → Underwritten	“Underwriting review”
Shopped → Accepted	“Prepare lender submission package”
D. Lender Interaction Triggers
Event	Task
Lender condition	“Clear lender condition #123”
Lender question	“Respond to lender inquiry”
Pricing expiration	“Re-lock or refresh pricing”

3️⃣ OWNERSHIP LOGIC (NO CONFUSION)

Ownership is role-based first, then user-based.

Example

“Request missing doc” → Processor role

“Income anomaly” → Underwriter role

“Borrower follow-up” → LO role

If multiple users exist:

Auto-assign based on:

Load balancing

Territory

Lender relationship

SLA history

📌 No shared responsibility. Ever.

4️⃣ SLA ENGINE (TIME IS ENFORCED BY CODE)
SLA Definition

Each task type has:

Target resolution time

Escalation path

Auto-actions on breach

Example SLA Table
Task Type	SLA	Escalation
Missing doc	48 hrs	Notify borrower → LO
Income review	24 hrs	Escalate to senior UW
Lender condition	12 hrs	Manager alert
Change-of-circ	Immediate	Freeze file
SLA States

Green: On track

Amber: 75% time elapsed

Red: Breached

📌 SLA color shows everywhere in the UI.

5️⃣ ESCALATION ENGINE (NO MANUAL FOLLOW-UPS)

Escalation is automatic and progressive.

Escalation Levels
Level	Action
0	Owner notified
1	Owner + Manager
2	Reassign task
3	Freeze loan state
4	Compliance alert

📌 Escalation actions are configurable without code.

6️⃣ AUTO-RESOLUTION (THE SYSTEM CLOSES TASKS)

Some tasks close themselves.

Examples

Borrower uploads requested doc → task auto-completed

Credit refreshed → review task auto-created, old closed

Lender clears condition → task auto-resolved

📌 Humans should never click “complete” if data proves completion.

7️⃣ TASK PRIORITIZATION (WHAT STAFF SEE FIRST)

SLA breach risk

Revenue impact

Borrower lock expiration

No custom sorting.
No personal preferences.

📌 This eliminates “I didn’t know what to work on.”
