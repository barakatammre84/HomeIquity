1️⃣ PRE-APPROVAL LETTER GENERATOR (SYSTEM DESIGN)
🎯 PURPOSE

Generate credible pre-approvals that:

Are trusted by real estate agents

Survive wholesale lender review

Do not expose you to lender liability

This is not a marketing letter. It is a controlled credit artifact.

A. PRE-APPROVAL LETTER DATA MODEL
PreApprovalLetter {
  letterId: string
  borrowerName: string
  loanAmount: number
  productType: "CONV" | "FHA" | "VA" | "HELOC"
  occupancy: "Primary" | "Second" | "Investment"
  expirationDate: Date
  underwritingSnapshotId: string
  conditions: string[]
  disclaimersVersion: string
  generatedAt: Date
  generatedBySystem: boolean
}


Key rule:

The letter references a frozen underwriting snapshot — never live data.

B. LETTER GENERATION LOGIC

Trigger conditions

Internal credit engine = APPROVABLE or APPROVABLE_WITH_CONDITIONS

All “hard stop” documents verified

Credit pulled with FCRA consent

Income classified with confidence ≥ threshold

System behavior

Auto-generate PDF

Apply watermark: “Pre-Approval — Subject to Lender Review”

Lock letter against edits

C. LETTER CONTENT (STANDARD STRUCTURE)
HEADER

Company legal name

NMLS ID

Contact info

Date

BODY (APPROVAL STATEMENT)

“Based on a preliminary review of the borrower’s credit, income, assets, and documentation provided as of the date below, [Borrower Name] has been pre-approved for a mortgage loan in the amount of up to $XXX,XXX, subject to the conditions listed herein and final lender underwriting.”

LOAN TERMS (NON-BINDING)

Loan amount

Loan type

Occupancy

Purpose

⚠️ Do NOT include rate unless locked

CONDITIONS SECTION

Auto-generated from rules engine:

Final lender underwriting approval

Satisfactory appraisal

Verification of unchanged financial condition

Additional documentation if requested

EXPIRATION

“This pre-approval is valid through [date] unless financial circumstances change.”

SIGNATURE

Automated system signature

Named LO (optional)

No underwriter attribution

2️⃣ EXACT BORROWER DISCLAIMERS (USE THESE)

These are intentionally conservative and widely accepted.

🔒 PRIMARY DISCLAIMER (NON-NEGOTIABLE)

“This pre-approval is not a commitment to lend. Final loan approval is subject to lender underwriting, verification of information provided, property review, and satisfaction of all loan conditions. Any change in credit, income, assets, or financial obligations may affect eligibility.”

🏦 BROKER ROLE DISCLOSURE

“We are acting as a mortgage broker and not as the lender. The final credit decision will be made by the selected wholesale lender.”

📄 DOCUMENT RELIANCE DISCLOSURE

“This determination is based on documentation and information provided by the borrower and third-party sources. Incomplete, inaccurate, or unverifiable information may result in additional conditions or withdrawal of this pre-approval.”

🔁 CHANGE-IN-CIRCUMSTANCE CLAUSE (VERY IMPORTANT)

“This pre-approval is contingent upon no material change to the borrower’s financial condition prior to loan closing.”

🧠 SYSTEM-GENERATED NOTICE (PROTECTS STAFF)

“This pre-approval was generated using automated underwriting analysis and is subject to human and lender review.”

3️⃣ HOW TO LEGALLY “FREEZE” DECISIONS (CRITICAL)

This is where most platforms fail.

You must freeze inputs, not outcomes.

A. UNDERWRITING SNAPSHOT (IMMUTABLE)

When pre-approval is issued:

UnderwritingSnapshot {
  snapshotId: string
  creditReportHash: string
  incomeMetrics: object
  assetMetrics: object
  dti: number
  dscr?: number
  ruleSetVersion: string
  createdAt: Date
}


🔒 Store as write-once
🔒 Hash all critical inputs
🔒 No recomputation without a new snapshot

B. DECISION FREEZE RULE

No recalculation unless one of these occurs:

Borrower uploads new document

Credit expires (typically 90–120 days)

Borrower reports change in employment, income, liabilities

LO explicitly requests re-analysis (logged)

Every re-run = new snapshot + new letter

C. VERSIONED DISCLAIMERS

Disclaimers are versioned artifacts.

DisclaimerVersion {
  version: "2026.01"
  text: string
  approvedByCounsel: boolean
}


The letter stores which disclaimer version was used.

This is massive for legal defense.

5️⃣ HARD RULES (DO NOT BREAK THESE)

❌ Never say “approved” without “subject to lender underwriting”
❌ Never change a pre-approval without generating a new snapshot
❌ Never let LOs edit letters manually
❌ Never backdate or overwrite snapshots
❌ Never hide conditions

These are the mistakes that get brokers sued.
